How to Run This Project

1
Hardware Setup
▼

Required Components:

  • Raspberry Pi 4 (4GB RAM recommended)
  • USB Microphone
  • 16x2 LCD Display with I2C Backpack
  • Servo Motor (SG90 or similar)
  • Jumper Wires & Breadboard
  • MicroSD Card (16GB+) with Raspberry Pi OS

Wiring Connections:

LCD Display (I2C): - VCC → 5V (Pin 2) - GND → Ground (Pin 6) - SDA → GPIO 2 (Pin 3) - SCL → GPIO 3 (Pin 5) Servo Motor: - Red (VCC) → 5V (Pin 4) - Brown (GND) → Ground (Pin 14) - Orange (Signal) → GPIO 18 (Pin 12) USB Microphone: - Connect to any USB port
2
Install Dependencies
▼

System Packages:

sudo apt-get update sudo apt-get install -y python3-pip portaudio19-dev libatlas-base-dev sudo apt-get install -y libjasper-dev libqtgui4 libqt4-test

Python Libraries:

pip3 install numpy sounddevice librosa pip3 install tflite-runtime RPi.GPIO pip3 install RPLCD smbus2

Note: Installation may take 15-20 minutes on Raspberry Pi.

3
Download Project Files
▼

Project Structure:

voice_door_project/ ├── 7882_project_final.py # Main production code ├── door_model.tflite # Trained TinyML model (117KB) ├── requirements.txt # Python dependencies └── README.md # Documentation

Download and Extract:

# Create project directory mkdir -p ~/voice_door_project cd ~/voice_door_project # Copy files from USB/download # Or clone from repository (if available)
4
Configure Hardware Settings
▼

Enable I2C Interface:

sudo raspi-config # Navigate to: Interface Options → I2C → Enable

Verify LCD Connection:

sudo i2cdetect -y 1 # Should show address 0x27 or 0x3f

Test Microphone:

arecord -l # List recording devices arecord -d 3 test.wav # Record 3 seconds aplay test.wav # Play back
5
Run the System
▼

Start Voice Control:

cd ~/voice_door_project python3 7882_project_final.py

Expected Output:

TinyML Voice Door System Ready Listening...

Usage:

  • Say "OPEN" clearly → Door opens (servo → 175°)
  • Say "CLOSE" clearly → Door closes (servo → 0°)
  • LCD shows status: READY / OPEN / CLOSED
  • System has 3-second cooldown between commands
  • Minimum 80% confidence required for execution

Stop System:

Press Ctrl+C to stop
6
Troubleshooting
▼

Common Issues & Solutions:

Problem: LCD shows nothing
  • Check I2C address: sudo i2cdetect -y 1
  • Verify wiring connections (SDA, SCL, VCC, GND)
  • Adjust LCD backpack potentiometer (contrast)
Problem: Microphone not detected
  • Run arecord -l to list devices
  • Replug USB microphone
  • Try different USB port
Problem: Servo doesn't move
  • Check GPIO 18 connection
  • Verify servo has power (5V)
  • Test servo separately with simple script
Problem: Low accuracy
  • Speak closer to microphone (15-20cm)
  • Reduce background noise
  • Speak clearly and distinctly
  • Check microphone sensitivity in alsamixer

Technical Specifications

Machine Learning Model:

  • Architecture: Depthwise Separable CNN
  • Parameters: 30,020 (29,380 trainable)
  • Input: 13 MFCC coefficients (39 with deltas)
  • Output: 4 classes (open, close, noise, unknown)
  • Framework: TensorFlow → TFLite conversion
  • Quantization: Float32 (117KB) / Int8 (46KB available)

Audio Processing:

  • Sample Rate: 16kHz (optimal for speech)
  • Window Size: 1 second
  • Feature Extraction: MFCC (Mel-Frequency Cepstral Coefficients)
  • Channels: Mono (single channel)

Safety Features:

  • Confidence Threshold: 80% (prevents false positives)
  • Cooldown Timer: 3 seconds (prevents spam triggers)
  • Error Handling: Graceful shutdown on Ctrl+C
  • Status Display: Real-time LCD feedback
